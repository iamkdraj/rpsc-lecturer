# UX Design: Challenge-First Learning App

## UX Direction
The app should feel like a premium interactive learning product: calm, focused, visual, and structured. It should not feel like a blog, notes PDF, internal experiment, or admin dashboard.

## Information Architecture
- Home: value proposition + sample challenge + course map preview.
- Courses: four tracks with outcomes and modules.
- Practice: lesson player.
- Mock Tests: diagnostic, section tests, full mock roadmap.
- Syllabus: complete official coverage map.
- Study Plan: sequence for exam preparation.

## Primary Components
1. Top navigation with app-like tabs.
2. Hero with a live challenge preview.
3. Course map node list / path.
4. Lesson sidebar showing current path.
5. Challenge card with:
   - concept card
   - example
   - trap
   - active question
   - options
   - explanation state
6. Progress/error summary card.

## Visual System
- Warm study background.
- White/paper cards.
- Strong blue primary action.
- Orange highlights for active/attention states.
- Green only for correctness/success.
- Rounded cards, generous spacing, high readability.

## Legal/Brand Guardrail
Use “challenge-first” and “interactive learning” patterns, but do not copy Brilliant.org’s exact layout, illustrations, colors, typography, wording, sequence, or trade dress.
