# PRD: RPSC Lecturer English Learning App

## Product Goal
Create a premium, learner-facing RPSC Lecturer English preparation app with an interactive learning experience inspired by challenge-first platforms, without copying any third-party UI, branding, or trade dress.

## User Problem
Aspirants have a large syllabus spanning grammar, literature, literary theory, pedagogy, and ICT. Most resources are notes dumps. Users need structured progression, active recall, MCQ practice, and confidence under negative marking.

## Product Principles
1. Practice before explanation: every lesson should make the learner think first.
2. Small steps: one concept per screen/challenge where possible.
3. Visible path: users should always know what to study next.
4. Exam discipline: show negative marking risk, confidence, and error review.
5. Trust: no fake claims, no public mention of AI-company experiment.

## Core User Flows
### Flow 1: New learner starts
Home → Course Map → First lesson → Challenge → Explanation → Next node.

### Flow 2: Targeted revision
Courses → Track → Topic cluster → Lesson/challenge → Error log.

### Flow 3: Mock preparation
Mock Tests → Diagnostic/section/full mock → Results → Weak topic recommendations.

### Flow 4: Syllabus confidence
Syllabus → Coverage status → Open missing/weak topic → Practice.

## MVP Requirements
- App shell with learner-facing navigation.
- Course map with nodes grouped by track.
- Lesson player with challenge-first layout.
- MCQ interactions with explanations.
- Syllabus coverage view.
- Mock test roadmap and later runnable mock engine.
- Study plan view.
- Static deploy to GitHub Pages.

## Non-Goals for Current Sprint
- Exact clone of any third-party site.
- Payments.
- User accounts.
- Backend database.

## Success Metrics
- User can understand the product in <10 seconds.
- User can start a lesson in <2 clicks.
- Build remains static and deployable.
- Every content item has answer explanations.
